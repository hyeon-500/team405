[백엔드 구조 개선 및 리팩토링]

수정일: 2026-06-08

1. WebSocket 모듈 분리

* app.js에 있던 Socket.IO 연결 관리 코드를 websocket/websocket.js로 이동
* 클라이언트 접속 및 해제 이벤트를 별도 모듈에서 관리하도록 변경
* broadcastSensorData() 함수 추가하여 실시간 데이터 전송 기능 분리

2. MQTT 처리 로직 정리

* mqtt_client.js의 역할을 MQTT 수신 및 데이터 흐름 제어 중심으로 정리
* WebSocket 전송 로직을 websocket.js로 분리

3. Service 계층 추가

* services/log_service.js 생성

  * sensor_logs 테이블 저장 기능 분리
* services/alert_service.js 생성

  * alert_events 테이블 저장 기능 분리
* services/api_service.js 생성

  * Flask AI 서버 통신 기능 분리
  * 기존 axios.js 파일명 변경

4. Database 구조 개선

* sqlite.js 위치 이동

  * 기존:
    backend/realtime_server/db/sqlite.js
  * 변경:
    backend/database/sqlite/sqlite.js

* SQLite DB 저장 위치 변경

  * 기존:
    realtime_server/log/sensor_monitoring.db
  * 변경:
    backend/database/sqlite/table/sensor_monitoring.db

5. app.js 정리

* 중복된 app.get('/') 코드 제거
* WebSocket 초기화 코드 모듈화
* 서버 시작 및 라우팅 중심 구조로 정리

6. 프로젝트 폴더 구조 개선

* database 폴더에 DB 관련 파일 통합
* services 폴더에 비즈니스 로직 통합
* websocket 폴더에 Socket.IO 관련 코드 통합

7. 불필요한 프론트엔드 파일 정리

* frontend/dashboard/package.json 삭제
* frontend/dashboard/package-lock.json 삭제
* frontend/dashboard/node_modules 삭제
* 백엔드 전용 패키지가 프론트엔드에 설치되어 있던 문제 해결
